## Tone of Voice

**Principles:**
- **"je" (niet "u"):** Toegankelijk, persoonlijk (niet afstandelijk formeel)
- **Bemoedigend:** "Goed bezig!", "Bijna!", niet "Fout." of "Wrong."
- **Context geven:** Leg "waarom" uit, niet alleen "wat"
- **Symbols:** ASCII brackets only (`[TIP]`, `[!]`, `[✓]`) — terminal aesthetic, NO emojis in code

### Sessie 157: `#23` validate-docs `--deep` mode — Soft-drift detectie (3 nieuwe checks: Bundle KB + Milestone-% + Cross-doc Versie) + REAL-TIME M8 drift caught via zelf-test (9 jun 2026)
⚠️ **Never:**
- `awk printf "%.1f"` als drift-percentage formatteren wanneer locale `nl_NL.UTF-8` actief is — Sessie 157 Phase C clean baseline gaf `awk: line 1: syntax error at or near ?` want `printf` produceert `0,0` (Nederlandse komma) en de volgende awk-call interpoleerde `(0,0 > 5.0) ? 0 : 1` = syntax-error. Erger: `if awk` interpreteerde fail als false → silent `pass` branch. Pre-data design: voor elke driftcalc in validate-script, gebruik pure-bash integer arithmetic (`(measured - target) * 1000 / target` voor 0,1% precision) ipv float-printf. Locale-onafhankelijk + geen interpolation-failure. Generaliseert naar elke shell-script die berekende getallen tussen tool-calls passeert
- Soft-drift checks (calibratie-gevoelig) verplicht maken in pre-commit hook — Sessie 157 `--deep` mode is bewust opt-in want bundle-KB ±5% tolerance is calibratie-gevoelig (na 2-3 sessies van groei kan styles/ breken zonder dat code wijzigt). Pre-commit hook moet fast-feedback geven; soft-drift behoort in `/summary` flow Step 7 als end-of-sessie validation gate. Verplichten = pre-commit friction zonder corresponderende fix-snelheid (geen quick-fix bij bundle-groei zonder code-aanpassing). Generaliseert naar elke validate-script die hard-drift en soft-drift mengt
- TODO-inline targeting expired sessie zonder fulfillment of expliciete deprecation — Sessie 157 verbouwde TODO Sessie 144 target (Sessie 140 commit, 13 sessies vertraagd) na 12 sessies vertraging. TODO werkte als forcing-function (niet vergeten) MAAR riskeert verkrachting van trigger-mechanisme als geen sessie het oppakt. Mitigatie: elk `/summary` flow stap 1 ground-truth meting MOET inline TODOs scannen op expired sessie-trigger. Als trigger-sessie > current-sessie - 5, expliciet AskUserQuestion of TODO doorgeschoven of opgepakt wordt. Voorkomt accidental indefinite-deferral

✅ **Always:**
- Pure-bash integer arithmetic met `× 1000 / target` voor 0,1% precision in driftcalc = locale-onafhankelijk forcing-function-discipline. Sessie 157 leerpunt: `delta=$((measured - target)); abs_delta=$(( delta < 0 ? -delta : delta )); abs_pct_x10=$(( abs_delta * 1000 / target ))`. Test `[ "$abs_pct_x10" -gt 50 ]` voor ±5% tolerance. Sign-handling via separate `sign` variabele. Generaliseert naar elke shell-script die percentage-comparisons doet zonder externe tools (awk/bc/python)
- `--deep` opt-in flag pattern voor validate-scripts die hard-drift (must-pass-always) en soft-drift (tolerance-gevoelig) mengen — Sessie 157 introduceerde voor `scripts/validate-docs.sh`. Pre-commit blijft fast (Checks 1-4 only), end-of-sessie `/summary` flow runs `--deep` (Checks 5-7) als forcing-function gate. Generaliseert naar elke validate-script die zowel binary invariants als calibratie-getoleerde drift checked. Pattern: `DEEP_MODE=0; for arg in "$@"; do [ "$arg" = "--deep" ] && DEEP_MODE=1; done` + `if [ "$DEEP_MODE" = "1" ]; then ... fi` wrapping
- Structured HTML-comment marker block als robust parse-target voor ground-truth-cijfers in docs — Sessie 157 TASKS.md `<!-- VALIDATE-BUNDLE-START -->` / `<!-- VALIDATE-BUNDLE-END -->` met `<!-- src=N styles=N blog=N assets=N -->` data-line. Documenteert drift-detection-target transparant + survives doc-rewrites + parse-robuust (`grep -A2 'VALIDATE-BUNDLE-START' | grep -oE 'src=[0-9]+...'`). Generaliseert naar elke validate-target waar ground-truth-cijfers in een doc geautomatiseerd gecheckt worden. Niet hergebruiken voor sessie-counter of versie (die hebben canonieke `**Bold:**` regel-prefix als anchor). Marker-block voor compound-data zoals 4-directory bundle KB

## Sessie Protocol

**Voor Sessie:** Lees `.claude/CLAUDE.md` (this file) + check sprint-regel + Volgende Stappen in `TASKS.md`
**Tijdens:** Markeer taken in `TASKS.md` direct | Architecturale beslissingen alleen in `PLANNING.md` bij echte arch-change
**Afsluiten:** Use `/summary` command → 6-step flow (zie hieronder)

### `/summary` flow — single source of truth = `TASKS.md`

1. **Ground truth meting** (~30 sec, read-only)
   - `du -sb src/ styles/ blog/ assets/` → bundle metrics
   - `find tests/e2e -name "*.spec.js" | wc -l` → test file count
   - `git log --oneline -1` → laatste commit voor sprint-regel

2. **Update `TASKS.md`** (primary execution-tracker)
   - Header: `Laatst bijgewerkt` datum + `Sprint` regel met huidige sessie
   - Footer: datum + version
   - Milestone-tabel: percentage update bij task completion
   - Bundle/test metrics: ground-truth getallen uit stap 1

3. **Update `docs/sessions/current.md`**
   - Volledige sessie-entry (mission, work done, learnings, next steps)
   - Rotation: bij Sessie %5 → archiveer pre-N-6 entries naar `archive-*.md`

4. **Update `.claude/CLAUDE.md`** (AI-context, lean — dit bestand)
   - "Recent Critical Learnings": prepend nieuwe sessie, behoud top 6, ouderen → `current.md`
   - "Sessie counter" regel
   - Footer datum + version
   - Live metrics in Quick Reference: **niet** updaten — verwijs naar TASKS.md

5. **Update `PLANNING.md`** ALLEEN bij architectuur-wijziging
   - Nieuwe tech-stack-keuze, design-system-change, security-strategie-shift
   - GEEN milestone-percentage-updates (woont in TASKS.md)

6. **Update `docs/prd.md`** ALLEEN bij scope-wijziging
   - Nieuwe requirements, success criteria change
   - GEEN tactical execution updates

7. **Validatie** (forcing function)
   - `bash scripts/validate-docs.sh` → exit 0 vereist
   - Pre-commit hook draait dit automatisch
   - Checks: sessie-counter alignment, datum-consistency binnen doc, PRD-version-match across docs

**Rotation trigger:** Every 5 sessions, archive sessies N-10..N-6 from CLAUDE.md learnings (last bulk: Sessie 145 archived 135-139, Sessie 146 1-in-1-out archived Sessie 140 → current.md, next bulk: Sessie 150)
